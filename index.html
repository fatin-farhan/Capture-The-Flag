Brute Force

Brute force is a way to guess a password or code by trying every possible combination until the right one is found. It is often used by hackers but can be stopped with strong passwords and security. Hint: Use Script

Challenge 2

XSS

XSS (Cross-Site Scripting) is a type of security attack where hackers insert harmful code into websites to steal information or control user actions. This can happen when websites don't properly check or clean user input. Hint: Source Code is Important

Goto CTF2

Challenge 3

CSRF

CSRF (Cross-Site Request Forgery) is a type of attack where a hacker tricks a user into doing something they didn't intend to, like changing their password or sending money, without their knowledge. Hint: Find login endpoints in source code

Goto CTF3

Challenge 4

Command Execution

Command Execution is a vulnerability where user input is executed as a system command. It occurs when functions like exec() or system() process unsanitized input. Hint: Username and Password from CTF1

Time-Based Blind SQLI

Time-Based Blind SQL Injection exploits response delays to infer database information. Attackers detect true/false conditions based on response time.
